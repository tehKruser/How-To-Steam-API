Welcome!

This tutorial was created for those of you who are interested in developing with the Steam Web API, but are unsure of how to get started. We'll walk you through an example of displaying Steam IDs, gamertags and recent games played. You'll see the JavaScript code for both the server and client side, and finally see that code in action.

Resources

Before we get started, you should review the following resources:

  • Host for your project: You'll need a place to host your website, whether it be on your home server, GitHub Pages, or any of the other countless options.

  • Steam Web API: Documentation on the list of available methods

  • Steam Web API Key: Obtain your API key for your project. The "Domain Name" is the base url of where your project is hosted.

  • Node.js: This tutorial will be using Node.js for JavaScript runtime.

  • AJAX: Asynchronous JavaScript and XML. Creates an object to communicate with server side scripts.

  • CORS: (Cross-Origin-Resource-Sharing) The Steam Web API does not allow AJAX requests from browsers due to security concerns of exposing the API key. This is an important concept to be familiar with, but if you don't completely understand it, do not worry. This tutorial should help you deal with it.

  • BreweryDB API Tutorial - Preparing a server: This is a great example of how to set up a server for back end requests to the Steam Web API. Follow this tutorial for how to set up the server, and then I'll show you how to modify the code to accept requests from your browser and then send those requests off to the Steam Web API.

Essentially, our communication will look like the following:

Browser ⇨ Server ⇨ Steam Web API ⇨ Server ⇨ Browser
